# NeuralVault API

Go backend for NeuralVault: source ingestion, embedding generation, workspace-scoped retrieval with streaming grounded answers, and OIDC authentication with tenant isolation.

## Commands

Run from this directory (or use the repo-root `Makefile` equivalents):

```bash
go run ./cmd/server           # start the API at :8080 (make run)
go test ./...                 # tests; add -race to match CI (make test)
golangci-lint run             # lint (make lint)
go run ./cmd/migrate up       # apply migrations (make migrate-up)
go run ./cmd/cli <args>       # nv CLI (make run-cli ARGS=...)
```

The API expects the infrastructure from the repo-root `docker-compose.yml` (Postgres, Qdrant, Ollama, MinIO, Keycloak) to be running — see [getting-started.md](../getting-started.md).

## Configuration

Config is read from `api/.env` (copy `api/.env.example`), validated at startup, and grouped by env prefix: `SERVER_`, `POSTGRES_`, `QDRANT_`, `OLLAMA_`, `MINIO_`, `AUTH_`. In non-production, `.env` is loaded first and `.env.<SERVER_ENV>` overrides it; in production only system env vars are used.

## Layout

```
api/
├── cmd/
│   ├── server/       # API entry point
│   ├── migrate/      # goose migration runner
│   └── cli/          # nv CLI — plain HTTP client, no internal/* imports
├── internal/
│   ├── <domain>/     # one package per feature domain:
│   │   ├── handler.go    # HTTP layer
│   │   ├── service.go    # business logic (interface + implementation)
│   │   └── routes.go     # Chi subrouter, mounted in router/router.go
│   └── ...           # config, storage, vectorstorage, objectstorage,
│                     # embedding, llm, chunking, model, logger, router
└── docs/             # generated Swagger output — do not edit by hand
```

Domains: `health` (public), `auth` (OIDC login + session), and the `RequireUser`-protected `workspaces`, `sources`, and `retrieval`.

## API docs (Swagger)

Served at `/swagger/` when the server is running. Generated by `swaggo/swag` from handler annotations — regenerate with `make swag` from the repo root after changing annotations. CI fails if `api/docs` is out of sync.
