# NeuralVault glossary

Canonical terms for this codebase. Grounded in `api/internal/model/`. Use these exact terms in code, comments, commit messages, issues, and PRs — do not introduce synonyms for concepts that already have a name here.

Agent-only reference: not linked from public docs. If a term needs to be explained to humans, it belongs in `docs/`, not here.

| Term | Meaning | Do not use |
|---|---|---|
| **Source** | A knowledge input a workspace ingests: `git`, `file`, or `web` (`model.SourceType`). Owns a lifecycle: `pending` → `indexing` → `indexed` / `error` (`model.SourceStatus`). | "Document", "connector", "input" |
| **Chunk** | A single indexed unit of content produced from a Source by the chunking engine, with a 1:1 mapping to a Qdrant point (`model.Chunk`). | "Segment", "fragment", "snippet" |
| **Workspace** | The tenancy boundary. Sources, chunks, and users are scoped to a workspace (`model.Workspace`, `WorkspaceID` on every owned entity). | "Project", "org", "team" (unless referring to the literal `WorkspaceRole` values) |
| **Ingest** | The service operation that (re-)triggers indexing of a Source (`Service.Ingest`, `POST /sources/{id}/ingest`). Distinct from "indexing", which is the resulting Source status. | "Sync", "process", "import" |
| **Splitter** | The interface that turns raw content into `Span`s for a given `ContentType` (markdown, plaintext) (`internal/chunking`). | "Parser", "tokenizer" |
| **Embedding** | The vector representation of a chunk's content, generated by the pluggable embedding provider (default: `nomic-embed-text` via Ollama). Not yet implemented — see roadmap Phase 1. | "Vectorization", "encoding" |
| **Provider** | A pluggable backend for embeddings or LLM inference (OpenAI, Claude, Gemini, Ollama, Qwen, DeepSeek). No provider is hardcoded. | "Model" (a provider offers models; the two are not interchangeable) |
| **User** / **UserIdentity** | `User` is the canonical account. `UserIdentity` links an external auth provider identity (`provider` + `external_id`) to a `User`. | Do not conflate — a `User` can have multiple `UserIdentity` rows |
| **Role** | A `UserWorkspace`'s permission level within a workspace: `owner`, `admin`, `member` (`model.WorkspaceRole`). | "Permission", "access level" |
| **Retrieval** | The (not-yet-implemented) query-time pipeline: semantic search → reranking → context compression, per `docs/architecture.md`. | "Search" alone (search is one step inside retrieval) |

## Adding a term

When a new domain concept is introduced (new `internal/model/*.go` type, new status/enum, new pipeline stage), add it here in the same PR. If a term already has an entry, reuse it — do not add a second name for the same concept.
